BVDK Scoreboard → OpenPowerlifting

Wandelt Ergebnisseiten von scoreboard.vportal-online.de in fertige meet-data/bvdk/-Beiträge um – inklusive Merge Request gegen openpowerlifting/opl-data. Alles läuft lokal im Browser.

First Step

Quelle laden

Die Ergebnisdaten stecken vollständig im HTML der Scoreboard-Eventseite – inklusive gültig/ungültig je Versuch.

Der Scoreboard-Server erlaubt keinen direkten Zugriff aus dem Browser (CORS). Das Tool probiert daher zuerst den Direktabruf und danach mehrere öffentliche Weiterleitungsdienste. Da diese Dienste von Dritten betrieben werden, läuft die (öffentliche) Ergebnisseite über deren Server – wenn das nicht erwünscht ist oder alle Dienste ausfallen, den Weg über die gespeicherte Datei nutzen.

Immer zuverlässig: Eventseite im Browser öffnen → Strg + S → Format „Webseite, nur HTML“ → gespeicherte Datei unten ablegen. Alternativ Strg + U (Quelltext anzeigen), alles kopieren und einfügen.

Second Step

Prüfen & ergänzen

Ort und Bundesland stehen nicht im Scoreboard – bitte ergänzen. Die Vorschau ist nach Gewichtsklassen gruppiert, Fehlversuche erscheinen rot.

Third Step

Dateien

Format wie in den bestehenden Meets (z. B. bvdk/2516): Fehlversuche negativ, Gäste als „G“, DQ ohne Total.

Für einen manuellen Merge Request: Ordner meet-data/bvdk/<Nr>/ im Fork anlegen und die drei Dateien einchecken.
Fourth Step

Merge Request erstellen

Erledigt automatisch: Fork (falls nötig), nächste freie Ordnernummer, Duplikat-Check, Branch, Commit, MR.

GitLab-Token erstellen – Schritt für Schritt
  1. Bei gitlab.com anmelden (kostenloses Konto genügt).
  2. Diese Seite öffnen: gitlab.com/-/user_settings/personal_access_tokens
    (oder: Avatar oben links → Edit profile → Access tokens)
  3. „Add new token“ wählen.
  4. Name vergeben, z. B. bvdk-opl-tool, und ein Ablaufdatum setzen (kurz halten, z. B. 1 Monat).
  5. Bei Scopes genau ein Häkchen: api.
  6. „Create personal access token“ klicken und den Token (glpat-…) sofort kopieren – er wird nur einmal angezeigt.
  7. Token unten einfügen. Fertig.

Der Token bleibt ausschließlich im Speicher dieser Seite und wird nur an gitlab.com gesendet – er wird nicht gespeichert. Nach getaner Arbeit kann er in den GitLab-Einstellungen jederzeit widerrufen werden (Revoke).
